About

  I am a hardware-oriented engineer specializing in digital RTL design, FPGA prototyping, and hardware security-centric system architecture. My work focuses on building correct-by-construction datapaths, tightly coupled PS–PL systems, and secure packet-processing pipelines on Xilinx SoCs.
  My technical foundation spans Verilog/SystemVerilog RTL, AXI-based SoC integration, embedded bare-metal software, and hardware/software co-design workflows. I have implemented and validated designs involving AXI-Stream packet pipelines, AXI-Lite control interfaces, DMA engines, and interrupt-driven Ethernet processing on Zynq and Kria platforms.
  A key area of interest in my work is hardware security—specifically secure Ethernet datapaths, entropy-based anomaly detection, and side-channel-aware hardware design. I place strong emphasis on timing closure, power-aware design decisions, and system-level validation using industry-standard EDA tools.

Projects

Ethernet-Based VLAN Switching on ARM Cortex-R5
Designed and implemented a bare-metal VLAN-aware Ethernet switching system on Xilinx Kria KV260. Developed packet classification, MAC table learning/aging, and VLAN-tag-based forwarding using PS GEM Ethernet and AXI-connected datapaths. Validated functionality and throughput using Wireshark.

Entropy-Driven FPGA Security Monitoring
Developed a real-time FPGA RTL engine to monitor entropy characteristics of network packet streams via AXI-Stream interfaces. Designed to detect anomalous traffic patterns directly in hardware, demonstrating the feasibility of on-chip security monitoring with minimal latency.

Scalable NoC Optimization for Chiplet-Based SoCs (2025)
Explored deadlock-free Network-on-Chip architectures for chiplet systems. Applied multi-objective optimization algorithms (SBPA, NSGA-II) to analyze bandwidth-latency trade-offs and improve routing scalability.

Low-Power FIR Filter Architecture: FPGA vs ASIC
Designed a 12-tap FIR filter using MATLAB/Simulink and synthesized the design on both FPGA and 90 nm ASIC. Performed detailed power, performance, and area (PPA) comparison using Cadence Genus, highlighting architectural trade-offs.
